## 3. Work／Codex開始条件

完全列挙・合理性NPC・方策比較・市場監査を開始できるのは、次のすべてがPASSした場合だけである。

```text
X0 = PASS
X1 = PASS
B1 = PASS
I1 = PASS
S0 = PASS
```

一つでも欠ける場合は、次の停止コードで止める。

```text
STOP_PREHUMAN_SIMULATION
```

B1／I1を行うための最小限の人間マイクロテスト器具は例外とする。情報OFF／ON人間マイクロテスト前に、完全列挙・合理性NPC・方策比較・市場監査へ進んではならない。

開始ACKの正本は `docs/info_game_knowledge_pack_v2/06_Work_Codex作業開始プロトコル_v2.0.md` とする。

## 4. 場面証拠と試行管理

- `DESIRED_SCENE`：設計前に望む場面。仮説であり、PASS証拠ではない
- `OBSERVED_SCENE`：人間マイクロテストで実際に起き、本人の言葉とともに記録された場面。S0の証拠に使える
- 試行番号は作品名・親系統名ではなく、中心動詞／対象／情報発生源／情報後の行動／利益経路からなる超系譜ハッシュで管理する
- 名前変更で試行番号をリセットしない
- 閉鎖済み超系譜の救済案と自己拘束型試行3を開始しない

## 5. 現在許可されている範囲

- 正式文書の機械的統合
- 人間マイクロテスト用の最小器具
- `X0/X1/B1/I1/S0` 全PASS後の完全列挙・構造監査

新作案の作成、閉鎖済み超系譜の再命名・救済、試行3の開始は許可されていない。
