# 【Claude Code 指示書】表層プロトタイプ ―― 交換台（情報層ゼロ）

発行日：2026-07-21 ／ 発行元：設計チャット4（Opus）
宛先：Claude Code
関連：新PJ_09（F-7・F-8）／実装知見（02）

---

## 【目的】

**「誰を繋ぎ、誰を待たせるか」だけで葛藤が立つかどうかを確認する。**

これは設立メモ §2-1「表層単体の面白さ ―― 情報層を外した表層が単体で面白くなければ kill」の検査である。
**面白さを証明するためのものではない。**設立メモ §3-1 の通り、ソロ／プロトタイプは
**ネガティブフィルタ（死んでいることの確認）にしか使えない。**

判定に使う言葉：**「どっちでもいい」「作業に感じる」「何も考えずに押している」が出たら死んでいる。**

---

## 【背景】

- 本作の表層は電話交換局（新PJ_09 F-8）。プレイヤーは交換手で、加入者からの呼び出しを繋ぐ
- 本番には情報層（秘匿・非同一性の証明）が乗るが、**このプロトタイプには一切乗せない**
- 情報層がゼロなので、**Firebase は不要。単一の静的HTML 1枚で完備する**（実装知見 §1-4）
- 本番テスターにはこれを触らせない運用のため、**1人で動かせること**が要件

---

## 【やること】

### 1. 成果物

- **単一HTMLファイル 1枚。**外部ファイル依存なし（CSS・JS はインライン）
- ファイル名：`switchboard_proto.html`
- 1人で最初から最後まで通せること（所要 5〜10分程度で1セッション終わる規模）

### 2. 画面

上から順に、次の3領域を持つ。**DOM は必ず分離すること**（後述の禁止事項参照）。

| 領域 | 内容 |
|---|---|
| **呼び出しキュー** | 待っている呼び出しの一覧。各行に「発信者名 → 相手の名前」「待ち時間（秒数がカウントアップ）」 |
| **交換台パネル** | プラグ3対。各対は「空き／通話中（残り時間つき）」を表示。通話中は誰と誰かを表示 |
| **記録欄** | 起きた出来事のログ（追記のみ・上が新しい）。ここがプレイヤーの唯一の成果表示 |

### 3. ルール（この通りに実装する）

1. 加入者は8〜12名。名前はシステムが固定で持つ（表示名のみ。属性や数値を持たせない）
2. 呼び出しは時間経過でキューに追加される（数秒〜十数秒に1件、ランダム間隔）
3. **プラグは3対のみ。**1つの通話が1対を占有する
4. プレイヤーは、キューの呼び出しを選んで空きプラグに割り当てる＝接続
5. 通話には長さがある（短い／長い）。**接続する前に長さは分からない**
6. 通話が終わるとプラグが空く。**その通話の帰結が記録欄に1行出る**
7. 呼び出しは待ち時間が一定を超えると**放棄される**（キューから消え、記録欄に「〇〇は諦めた」と出る）
8. **帰結は数値スコアではなく、出来事の文である**（F-7 帰結型）。良い／悪いの明示的な評価を出さない
9. **一部の帰結が、後続の呼び出しを変える。**（例：ある通話が成立すると、数十秒後に別の呼び出しが新たに発生する。ある呼び出しを放棄すると、以後その加入者からの呼び出しが来なくなる）
10. セッションは固定時間（例：5分）で終了。終了時に記録欄の全文を表示して終わる

> **9 が本プロトタイプの本体である。**これが無いと単なる待ち行列の処理（最適化型）になり、
> F-7 に違反したものを検査してしまう。**帰結が後続を変えることが、判断を生むかどうかを見ている。**

### 4. 数値パラメータ

**すべて画面上のスライダーまたは入力欄で、実行中に変更できるようにすること。**
（プラグ本数／呼び出しの発生間隔／放棄までの待ち時間／通話の長短の比率／セッション長）

理由：机上では適正値が決められないため、触りながら探す。**再読み込みなしで変えられること。**

### 5. 実装上の必須事項（実装知見 02 より。省略不可）

- **`confirm()` / `alert()` / `prompt()` を使わない**（§3-2。埋め込み環境で黙って false を返す）。
  確認が要る操作は2段階クリック（1回目で文言と色が変わり、数秒で自動キャンセル）
- **全体 `innerHTML` の再描画をしない**（§3-3）。
  リアルタイム更新される領域（キュー・パネル）と、スクロール位置を持つ領域（記録欄）の**DOM を分離する。**
  記録欄は追記のみで、既存行を書き換えない
- **Tailwind の任意値クラス（`bg-[#...]`）を使わない**（§3-4）。色は CSS 変数で一元管理する
- **エラーを握りつぶさない**（§2-6）。画面上の固定領域にエラーを表示する
- 非同期処理（setTimeout / setInterval）の入口に必ずガードを入れる（§3-2 関連バグ）
- **画面に VER 定数のバッジを出す**（§5。キャッシュによる「更新したのに反映されない」対策）
- 効果音は**任意**。付ける場合は Web Audio の合成音のみ（§4-1）。ミュート切替を必ず用意する

## 【触ってはいけないもの】

- **Firebase・オンライン同期・認証を一切実装しない。**このプロトタイプに秘匿情報は存在しない
- **情報層の要素を一切入れない。**具体的には以下を実装しないこと：
  - 加入者の「正体」「隠された属性」
  - 同じ名前が2つある／別の局が存在する、といった仕掛け
  - 不一致・非同一・確約・署名にあたる機構
  - **同じ加入者を両端に差す操作**（本番の核。プロトタイプに入れると初見が焼ける）
- **メモ機能・呼び出し一覧の自動整理・並べ替え・フィルタを実装しない**（handoff 04 §1-1 で凍結済みの禁止事項）
- **スコア・評価・「上手い／下手」の表示を作らない。**帰結は出来事の文だけで表す（F-7）
- 既存の `index.html` をリネームしない／既存ファイルを書き換えない（§5）
