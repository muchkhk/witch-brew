## 5. 判断を要する箇所（統合の際に、指示書だけでは決められなかった点）

以下は勝手に決めず、現在地ファイル内に `<!-- 要判断: ... -->` のHTMLコメントで印を残した。

1. **§1 の表セル内での改行の扱い。** 指示書は §1 を「表形式」と明示的に指定しているが、
   F-1・F-4・F-7・F-8・F-9・F-11・F-17 の凍結文は原本で複数行の引用になっており、
   Markdownの表セルには改行をそのまま書けない。本ファイルでは原本の改行を `<br>` に
   置換して収めた（文言・強調記号・語順は一切変えていない）。この置換ルールで良いか、
   あるいは表形式自体を諦めて別の提示形式（見出し＋blockquoteの繰り返し等）にするかは、
   指示書だけでは決められなかった。現在地ファイル冒頭に注記とHTMLコメントを残した

2. **04 §1-4【凍結・設計要件】の扱い。** 「断片は、口頭で自然に報告した時点では整合して聞こえ、
   形式的な確約を切る瞬間に初めて矛盾が顕在化する形でなければならない」という凍結文は、
   F番号を持たないにもかかわらず、05§2-2・09§2-2（基準6）で「今も要求される」と
   繰り返し参照されている、現役の凍結要件と見られる。しかし現在地ファイルの指定構造
   （F-1〜F-17の表）はF番号を持つ項目専用であり、この項目を追加のF番号として登録するのか、
   それとも別枠を設けるのかは、指示書の【触ってはいけないもの】「凍結文を1文字も変えず
   逐語でコピーする」「新しい設計項目を書き加えない」という制約と、この項目自体は
   新規ではなく既存の凍結文であるという事実との間で、Claude Codeの裁量では決められなかった。
   §1 の表には含めず、出典対応表で孤児として記録するに留めた

3. **§6「規律の記録」の統合元に `新PJ_09` §8 を含めるかどうか。** 指示書 §2-1 の章立ては
   統合元を「04§7・05§4・07・12§8」と明示しており、09は挙げられていない。09§8は
   他の4節と同種の内容（役割分離・判断種別の記録・保留項目の明示・凍結テキストの受け渡し方法）
   を含んでおり、単純な書き漏らしである可能性がある一方、意図的な除外（09は別系譜のチャット
   という位置づけのため）である可能性も否定できない。指示書の指定を字義通り守り、09§8は
   統合せず孤児として記録した

4. **F-2・F-3 が単一の凍結文を共有している点をどう表に反映するか。** 05の原表は両者を別の
   F番号として立てているため、現在地ファイルでも別行として維持し、同一の引用文を両方に
   転記した。1つの凍結文に2つのF番号を与えるという原本の構造自体の是非は判断せず、
   そのまま踏襲した

5. **台帳（§5）に「解消済みの懸念」を載せるかどうか。** 04§5-2の3つの懸念（テーマ融合・
   反転の言語化・人数）は05§1-1で解消・緩和と評価されているが、これらは「埋葬・破棄・
   保留・降格」のいずれにも厳密には当てはまらない（生きたまま解消された経緯）。
   指示書が台帳に明示的に要求する7項目には含まれていないため、台帳には追加せず、
   出典対応表の説明にとどめた。これで十分かは判断を仰ぐ

## 6. 作業中に設計判断を行っていないことの自己申告

指示書 §0-2 の禁止事項1〜6について、それぞれ次の通り申告する。

1. **凍結文を1文字も変えず逐語でコピーすること。** 遵守した。§2「機械的な検証」に記載の
   通り、F-1〜F-17・K-1〜K-6を含む全ての凍結文をPythonスクリプトで原本と突き合わせ、
   完全一致を確認済み（V3）。ただし表セルへ収めるための改行→`<br>`置換を行った点は
   §5-1 に判断を要する箇所として記録した。これは文字の変更ではなく改行コードの表現方法の
   変更であるという理解だが、この理解自体が正しいかは指示書だけでは確認できない

2. **矛盾を見つけても自分で解決しないこと。** 遵守した。§4 に列挙した6件の矛盾・不整合は、
   いずれも解決を試みず、事実として記録するに留めた（項目1「プラグ本数の競合」のみ、
   本統合作業より前・switchboard_proto.html実装時に既に解決済みだった事実を記録している）

3. **「これは不要そう」「これは古い」と判断して内容を削らないこと。** 遵守した。孤児
   （§3・15件）は一つも削除せず、全件を現在地ファイル §7 と本報告書 §3 の両方に記録した。
   「対応なし」という記載は削除ではなく、受け皿がないという事実の記録である

4. **表現の改善・誤字修正・体裁の統一を行わないこと。** 遵守した。原本（00〜12）には
   一切の編集を加えていない（コピー後・移動後の両方でDownloadsの原本とバイト単位一致を
   確認済み）。現在地ファイル・報告書・archive/README.md は新規作成物であり、これらの
   文章表現（見出し・注記・要判断コメント等）はClaude Codeが本指示書の指示に従って
   新規に書いたものである

5. **新しい設計項目・提案・改善案を書き加えないこと。** 遵守した。F-17は指示書の附録Aから
   逐語コピーしたものであり、Claude Code が新規に考案した設計項目は無い。ただし、
   §5-2 に記載した通り「04§1-4の扱い」を巡って、この項目をF番号として追加登録すべきか
   という論点自体は本作業中に発見したものであり、これは新しい設計項目の提案ではなく
   「既存の凍結文に受け皿がない」という構造上の発見である、と理解している

6. **元ファイルを削除・上書きしないこと。移動とコピーのみ。** 遵守した。00〜03は
   移動していない（内容・場所ともに変更なし）。04・05・06・07・09・10・11・12は
   `docs/新PJ/` から `docs/新PJ/archive/` へ移動した（コピーではなく移動。移動元には
   残っていない）。8件とも移動後にDownloadsの原本とバイト単位で一致することを確認した

**言えないものがあれば正直に書く、との指示に対して：** 上記6項目について、明確な違反は
無いと考えるが、1（改行の`<br>`置換）と5（04§1-4の扱いの発見）の2点は、境界線上の
判断であり、完全に「何もしていない」とは言い切れない。設計チャット側での確認を求める。
