## 第1部　実際に殺した死因（全件）

### 1-1. 帳簿づけへの堕落 ★新PJで最も再発しやすい

| | |
|---|---|
| **出所** | 伝達核v0.1（順列の秘匿伝達） |
| **症状** | 秘密が「順列」だったため、プレイヤーの主要な作業が**記憶と照合の管理**になった。読み合いではなく事務処理 |
| **診断** | 秘密の形が複雑（順列・写像・多次元）だと、それを整理する作業が中心に来る |

> **新PJでの再発形：**設立メモ §1-2 で既に警告した通り。
> **「断片A＋断片B＝答え」は、まさにこの死因である。**
> 情報を持ち寄って表を埋める作業は、反転ではなくパズルの組み立て。
> **反転が成立するのは、合流によって「既に見たもの」の意味が変わるときだけ。**

**検査法：**プレイヤーが紙にメモを取り始めたとき、そのメモは
「気づきの補助」か、それとも「システムが管理すべきものの代行」か。後者なら死んでいる。

### 1-2. 確率計算・電卓化への堕落

| | |
|---|---|
| **出所** | ブラフ核v0（終盤がただの確率計算に）／『遅信』（上級者が電卓的に最適化）／統合知見「一意解型」 |
| **症状** | 読み合い・推理に見えて、実際は計算すれば答えが出る。**複雑な計算があっても、最終的に毎回同じ答えなら作業** |
| **診断** | 状態空間が小さいと、時間さえかければ全部解ける |

> **新PJでの再発形：総当たり。**手がかりの組み合わせが少なければ、
> プレイヤーは推理せず全パターンを試す。「時間はかかるが必ず解ける」は、
> **不確実性があるように見えて一意解型**である。
> → 机上プレフライト検査1「総当たり耐性」／一発kill条件「総当たり検出」がこの検出器。

### 1-3. 生命感の喪失（賭けたものが軽い）

| | |
|---|---|
| **出所** | ブラフ核v0（チップ＝命が名目だけ）／伝達核v0.1（ゼロサム経済） |
| **症状** | リソースを賭けているのに、失う痛みがない |
| **治療（実証済み）** | **燃焼式経済** ―― 賭けたものは戻らない。ゼロサムだと「誰かに移っただけ」で軽くなる |

> **新PJでの再発形：**「観測機会を1つ使った」が痛くない設計。
> 観測が実質無制限なら、配分に緊張が生まれない。**有限化は必要条件だが十分条件ではない**（→ 1-4）。

---

### 1-4. 緊張の不在 ―― ドラマ原理の不発 ★最重要

| | |
|---|---|
| **出所** | フリーサイコロ（死因5・正式埋葬） |
| **経過** | 不可逆・罰する未来・正しい自己帰属を**すべて建てた**。詰み手番は36→3131に激増し、罰は機械的に完全に機能した。**それでも面白くなかった** |

> **死因：後悔の強度は、罰の大きさではなく、決断の瞬間に感じた緊張に比例する。**
> 決断時に緊張が無ければ、罰は「後悔」ではなくただの「結果」として処理される。
> **「そこまで考えなかった」選択は、どれだけ痛い帰結を生んでもドラマにならない。**

**緊張が生まれなかった理由2つ（そのまま検査項目になる）：**

1. **貪欲な現在手が、常に利用可能・十分・認知的に安価だった。**
   未来は存在したが、現在の決断に割り込む強制力を持たなかった。考えなくても打てて、考えなくてもそこそこ勝てる
2. **賭け金が分散していた。**1回の決定が背負う重みが小さく、代替が利く。
   **緊張は賭け金の集中から生まれる**（対比：影の競りは緊張を「入札の瞬間」の一点に集中させていた）

> **運用：「不可逆」系の核を検討するときは、罰の設計より先に、**
> **決断の瞬間に緊張を強制する仕組み（現在手の否定・賭け金の集中・時間圧）があるかを確認する。**

**新PJでの再発形：**「一回きり・不可逆」は核だが、**不可逆であることは緊張を保証しない。**
確約を切る瞬間に、他の選択肢が実際に魅力的でなければ、不可逆は形式だけになる。

### 1-5. 無料の推定が、有料の事実を駆逐する（市場不成立）

| | |
|---|---|
| **出所** | 思考課金ポーカー（死因6）／案D（覗き市場） |
| **症状** | 情報を買う仕組みを作ったが、**タダで推定した方が十分**なので誰も買わない |

> **市場原理：情報に値付けする前に、**
> **「無料推定と真実のギャップ × それに基づく行動の利得 > コスト」を確認せよ。**

**新PJでの再発形：**ヒント・追加観測にコストを設定する場合、
**そのコストを払わなくても十分に進めるなら、コストは飾りになる。**
逆に、払わないと進めないなら、それはコストではなく手順である。

---

### 1-6. タダ乗り ―― 情報を共有すると調査が死ぬ

| | |
|---|---|
| **出所** | 『影の競り』（死因2・最重要と記録） |
| **症状** | 調べた結果を他人に漏らすと、**調べない方が最適**になり、調査行動そのものが死ぬ |

> **新PJでの再発形：協力ゲームなので、これは構造的に発生する。**
> 全員が観測結果を共有するなら、**「自分が観測する」インセンティブは
> チーム全体の必要性だけになり、個人の判断が消える**（＝奉行問題の別表現）。
> **設立メモ §1-2「誰も全体を見ていない」構造が、この死因の対処でもある。**

---

### 1-7. 覗き支配 ―― 情報アクセスが強すぎて体験が崩壊する

| | |
|---|---|
| **出所** | 案D（シュレディンガー×ロンデル・完全埋葬） |
| **症状** | 情報を得る行動が支配的に強く、他の全ての行動が意味を失う |

> **新PJでの再発形：**「もう1回観測する」が常に最善なら、配分の判断は存在しない。
> **観測のコストが、観測の価値に釣り合っているか。**

---

### 1-8. 終わり方の不在 → 目的の後付け

| | |
|---|---|
| **出所** | 候補6（開始前埋葬） |
| **症状** | 情報機構は完成したが、**終わり方が無い**。そこから逆算して目的を後付けした |

> **新PJの歯止め（設立メモ §2-3）：情報機構を先に作ってよい。ただし**
> **「その情報を知ったプレイヤーが、次に何をするか」が空欄のまま先へ進まない。**

### 1-9. 全面非公開の中では、一点の公開シグナルが機能しない

| | |
|---|---|
| **出所** | 案A『見なかったことの証明』 |
| **症状** | 情報環境が真っ暗な中に一点だけ光を置いても、受信側が解釈できない |

> **一般化：メカニクスには相性の良い情報環境がある。**
> 同じ仕掛けでも、周囲の情報環境が違えば機能しない。

**新PJでの再発形：**手がかりを1つだけ丁寧に作っても、
**周囲の情報環境（何が既知で何が未知か）が定まっていなければ、その手がかりは読めない。**

---

### 1-10. ブラフ価値の従属性

> **フェイクは、本物のシグナルが処理されているときにのみ機能する。**
> 受信側が壊れていれば、発信側も死ぬ。

**新PJでの再発形：**「ミスリード」「叙述トリック」は、
**正しい読みが成立している場合にのみ機能する。**
プレイヤーが手がかりを正しく読める土台が無いところに嘘を仕込んでも、ただのノイズになる。

---

### 1-11. 学習崩壊・初回の偽陽性

| | |
|---|---|
| **出所** | フリーサイコロ（骨だけ版で「指が止まる」を確認 → **後に初回の新規性による偽陽性と判明**） |

> **初回の迷いは、理解不足でも発生する。**
> **「初回に指が止まった」だけでは、核の証明にならない。**

**新PJでの重大な含意 ―― これは構造的な問題：**
一回きりのゲームは**初回しか存在しない。**
つまり**偽陽性を反復プレイで剥がす手段が、原理的に無い。**

> **対処：「面白かった」という感想を証拠にしない。**
> 一発kill条件（特に**エウレカの不在**：正解の理由を自分の言葉で説明できるか）で判定する。
> 新規性による興奮と、気づきによる興奮は、**理由を説明させれば区別できる。**

### 1-13. メタ適応による反転の空転

| | |
|---|---|
| **出所** | 新PJ 設計チャット10（机上導出。実プレイ未検証） |
| **症状** | 反転を繰り返すと、プレイヤーが「何も信じない」という構えを獲得し、
　　　　　　 以後の反転が事前に無効化される |
| **診断** | 反転の連鎖は、答えが反転する連鎖ではなく、疑う場所が反転する連鎖である。
　　　　　　 各反転は、前の反転で「疑わなくてよくなった場所」を破らなければならない |

> **原理：反転の個数には上限がある。連鎖は種類の階段でなければならない。**
> 同じ種類の反転は、2度目から作業になる（類型25 学習崩壊型の変種）。

**現時点の机上判断：N=3。**最後の1つは「何も信じない」構えでは防げない種類でなければ
ならない。候補は「疑わせていたものが、実は文字通り本当だった」型、または目的側の反転。
ただし目的の反転は死因1-8（終わり方の不在）に直結するため扱いは慎重に。

**判断種別：机上判断（Opus）。実プレイ未検証。N=3 は根拠の弱い暫定値であり、
検証手段が現時点で存在しないことを明記する。**

### 1-12. プロセス上の死因（設計運用そのものの失敗）

| # | 内容 |
|---|---|
| **対症療法の連鎖** | 個々の修正は合理的でも、積み重なるとシステムが複雑化し「そもそも何が問題か」の診断が不能になる。**3つ溜まったら実装を止めて構造へ戻る** |
| **NPCソロテストの誤用** | 機械的NPCでは「人間同士の心理」を検証できない。**バランス崩壊のチェック用と割り切り、面白さの判断には使わない** |
| **机上分類の不信** | 「これは対症療法だろう」という当初の仕分けが、実際に測ったら間違っていたケースが複数回。**推測と検証を分けて扱う** |
| **保留にした論点ほど核心に近い** | 一度「後回しでいい」と思った論点を早めに深掘りすると、致命的な誤りが早期発見できる |
| **僅差killの敗者復活をしない** | 「実質合格では」が事後的な閾値再解釈の入口。僅差だったという**観察**は記録する。**行動**（復活）には変換しない |
| **物差しの検品を先に置く** | 判定指標を作ったら、**既知の死体にそれを当てて「死体が受かってしまわないか」を最初に確かめる** |
| **賭けの一文の不在** | 「このゲームの面白さは◯◯である」という陽性の命題を、作家の言葉で一文にできない状態で、構造の精緻化を進めない。陰性フィルタ（kill条件・検出器）をいくら通しても、賭けの不在は検出されない。フィルタは死を検出する装置であり、生を証明する装置ではない。**AIによる代筆は禁止**（フィルタを生成器に使う失敗〔設立メモ §0・D-2〕の再演のため） |

## 第2部　失敗の9類型（診断チェックリスト）

案を作ったら、**必ず全項目に当てる。**

| # | 類型 | 症状（一文） | 検査 |
|---|---|---|---|
| **19** | **目的不明型** | 何をしたいゲームなのか分からない | 何を目指すか／何を見て判断するか／何が上手いプレイか／どの瞬間を楽しむか ―― 4つとも言えるか |
| **20** | **一意解型** | 計算すれば決まる | 期待値最大の行動が常に同じでないか。**複雑な計算があっても最終的に毎回同じ答えなら作業** |
| **21** | **無差別型** | どちらでもいい | 得点差がほぼない／後続状況が変わらない／結局同じ状態へ収束する |
| **22** | **博打型** | 結局は運 | 結果差は大きいが判断材料がない。**「正解がない」のではなく「判断が成立していない」** |
| **23** | **逆インセンティブ型** | 核となる行動を取らない方が強い | 探索させたいのに探索しない方が効率的／情報を買わせたいのに無料推定で十分。**数値調整では直らないことが多い** |
| **24** | **変換消失型** | 合っていたのに意味がない | 推定精度は上がり行動も変わるが、利益や満足につながらない。**判断の存在と、判断が報われることを分けて検証する** |
| **25** | **学習崩壊型** | 一度遊べば十分 | 初回の驚きが消えると何も残らない／上達すると考える必要が減る |
| **26** | **前提不成立型** | その行動は、想定した情報を伝えていない | 秘密が別の数値から完全に逆算される／核と土台の情報環境が矛盾する。**この層は調整では直らない。核・土台・情報構造の交換が必要** |
| **27** | **対症療法肥大型** | 問題が出るたびに足す | 補助・特殊能力・救済ルール・例外処理。**3つ重なったら、元の核が弱い可能性が高い** |

**新PJで特に危ないもの：**

- **20 一意解型** ―― 総当たりで解ける謎はここ
- **25 学習崩壊型** ―― 一回きりなので反復では検出できない。**エウレカの不在で代替検出する**
- **26 前提不成立型** ―― 謎が「解けるはず」の論理的前提（手がかりから答えへの導出経路の実在）が成立しているか。**機械的検証が可能な領域**（実装知見 §8）

## 第3部　二原理と、利得内蔵シグナル

### 3-1. 二原理（対になっている）

> **シグナルの原理：コストを払う行為だけが、選好を伝える**（情報の側）
> **ドラマの原理：緊張を払った決断だけが、後悔を生む**（感情の側）
>
> **両方を満たして初めて、決断は「重く」なる。**

**新PJでの読み替え：**

| 原理 | 新PJでの形 |
|---|---|
| シグナル原理 | **観測の配分そのものが、何を重視しているかを示す。**何を見たか＝何を捨てたか |
| ドラマ原理 | **確約を切る瞬間に緊張が無ければ、外しても後悔は生まれない**（→ 死因1-4） |

### 3-2. 利得内蔵シグナル（旧PJの中心的な発見）

> **シグナルとなる行動には、本人にとっての実利を内蔵させる。**
> 「情報を伝えるためだけの行動」は、コストが説明されず、嘘も本当も等価になる。

『魔女の調合』で成立した構造：ある行動が、**本人にとっては得点を取る手であり、
同時に他者にとっては選好の手がかりになる。**この二重性が鍵。

**成立条件（7項目）：**

| # | 条件 | 内容 |
|---|---|---|
| 1 | **行動単体の実利** | その行動は、情報を無視しても本人に得がある |
| 2 | **機会費用** | 他を諦めて選んでいる（＝選好が乗る） |
| 3 | **秘密との因果関係** | 秘密の内容が、行動の選択を実際に変える |
| 4 | **複数解釈** | 一意に読めない（読めれば推理ではなく計算） |
| 5 | **行動変更への変換** | 読んだ側の行動が実際に変わる |
| 6 | **利益への変換** | その行動変更が利益になる（→ 24 変換消失型の防止） |
| 7 | **答え合わせ** | 最後に真偽が分かる |

> **新PJでの適用範囲の注意：これは対人シグナルの原則である。**
> 新PJの本体は対システム（世界の法則・仕込まれた事実）の解読なので、
> **1〜7をそのまま適用する場面は限られる。**
>
> **ただし 4・5・6・7 は対システムでもそのまま効く：**
> - 4 複数解釈 ―― 手がかりが一意に読めたら推理ではなく作業
> - 5 行動変更 ―― 気づいても行動が変わらないなら意味がない
> - 6 利益変換 ―― 気づきが結末に効かないなら変換消失型
> - 7 **答え合わせ** ―― **エウレカの成立には検証可能性が必須**

### 3-3. 即時停止条件

以下に該当したら、その場で設計を止めて構造へ戻る：

- 情報を得ても、行動が変わらない
- 情報を得なくても、十分に勝てる
- 情報が一意に読める（＝計算）
- シグナルとなる行動に、本人の実利が無い
- 「今回はブラフが起こるはず」と人物性を外部に期待している
