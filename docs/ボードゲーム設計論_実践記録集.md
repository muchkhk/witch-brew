案Aは最終的に「完成した名作」には至らず、主軸(インカの黄金型)を一旦中止した。
だが**この過程そのものが、方法論編の「アイデア1割・調整9割」を最も生々しく実演した記録**であり、

## 1. 出発点:案Aの核とは何だったか

**案A「見なかったことの証明(Committed Blindness)」の当初の核は2要素あった:**
- **要素1:「見る/見ない」の選択そのもの**(確実性を取るか、賭けるか)
- **要素2:「見なかったこと」をシステムが証明・保証する**(デジタルだからこそ「覗いていない」を担保でき、盲目の決断が能動的戦略になる=方法論の「デジタルネイティブの鉱脈」)

方法論編の検査(3試案の比較)で、案Aは最有力と判定された。理由:
- ジレンマ(§2-4):推論への賭け ◎
- 敗北感(§4-1):帰属(自分の賭け)と緩衝(見なかった逃げ場)の**両立** ◎(3案中唯一)
- AP抑制(§3-2):見ない=分析不能で構造的に長考を封じる ◎
- 穴が既知技法で塞げる ◎

## 2. 土台選び:足し算 vs 掛け算(方法論編§6の実例)

案Aの核「見る/見ない=リスクを取るか避けるか」を最も活かす土台を検討。
- セットコレクション(足し算=別の面白さを追加)→ 核が脇役になりがち
- **プレス・ユア・ラック(掛け算=同じ「リスク」主題を増幅)→ 核が主役に** ← 採用

→ **教訓:核を主役にしたいなら、核と同じ主題を扱う土台(掛け算)を選ぶ。** 案Aとインカ型PYLは「リスクとの付き合い方」を共有する。

## 3. 変形の軌跡 ― 核を巡る往復(記録の核心)

仮ルールを具体化するたびに穴が出て、潰すたびに核を見失いかけ、設計者の差し戻しで引き戻された。
この往復のパターンこそ再利用可能。

### 往復1:上位互換の穴
- **問題:** 「見て降りる」が「見ないで降りる」の上位互換に(降りる局面では見て危険回避が常に得)。
- **誤った対策:** 「潜る=必ず盲目」に→ **核(見る/見ないの選択)を殺した。**
- **正しい対策:** 「見る」の利点を"危険回避"から**"価値確定"**に付け替える。見ても呪物は見分けられない設定にすれば、降りる局面でも「確実な等倍 vs 賭けたまま高倍率」の真の二択が残る。核を保持したまま穴だけ塞げる。
- **教訓:穴を塞ぐとき、修正が長所(核)を殺していないか常に問う(方法論§3)。** 塞ぐべき一点だけを外科的に潰す。

### 往復2:同時性 vs 反応の駆け引き
- **問題:** 同時公開だと「他人が降りたのを見てから自分が判断」ができない。
- **対策:** 一潜航を**複数の同時ラウンド**で構成。各ラウンドは同時公開(ダウンタイムなし§1)だが、ラウンド間で前ラウンドの浮上者が判明→反応の連鎖が作れる(インカの黄金の構造)。
- **教訓:同時性(ダウンタイム解消)と逐次性(反応の駆け引き)はトレードオフ。ラウンド分割で両取りに近づける。**

### 往復3:即死バーストの矛盾
- **問題:** 「危険を抱えたまま迷う」緊張を謳ったのに「危険=即バースト」では、掴んだ瞬間に終わり、迷う局面が存在しない。
- **対策:** 危険を**即死でなく累積型**(閾値でバースト)に。見ないで抱えると自分の危険蓄積が不確実になり「もう一段いくか」の緊張が生まれる。
- **教訓:謳った体験(抱えたまま迷う)と、罰の即時性が矛盾していないか点検する。**

### 往復4:倍率のフレーバーと「見る」の死(二重の指摘)
- **問題A:** 「見ないで拾う=高倍率」のフレーバー的理屈が素直に出ない。
- **問題B:** ノルマ等がないと、見ないが常に得で「見る」が死んだ選択肢(§2-4偽の選択肢)になる。
- **根の共通性:** 「見ないの高倍率」という前提自体が無理をしていた。
- **対策:** ①目標(ノルマ)を導入し「見る=狙った値を確実に」に存在意義を与える。②倍率を捨て、見ない=「速い/安い/ブレる」、見る=「遅いが確実」に。フレーバーも「封を解くと目減り/封じたまま漁ると手早いがブレる」で素直に。
- **教訓:調整には「最初の核の一部(倍率)を捨てる」ことも含まれる。案Aの真の核は「見る/見ないの選択」であって「高倍率」ではなかった。** 前提を疑う勇気。

### 往復5:核(要素2)の喪失と回収
- **問題:** 変形の過程で「見る/見ない」を私的な効率選択に閉じ込め、**要素2(見なかったことの証明)を殺していた。** 私的な盲目引きは物理のバッグビルドでも可能で、デジタル固有の新規性が消えていた。
- **対策(概念上):** 「見ずに取った事実を公開し、システムが保証する」→ 豪胆さのシグナル/ブラフ(§5-4)として機能させる。
- **教訓:要素2が輝くのは「他者に対して意味を持つ」とき。私的選択に閉じると死ぬ。**

## 4. 競争構造の欠落(§1への差し戻し)

- **問題:** ここまで「一人の潜航をどう面白くするか」に集中し、**プレイヤー同士が何を競うか(§1)が未定義**だった=マルチプレイヤー・ソリティア(§0)。
- **原因:** 魅力的な核(見る/見ない)に引っ張られ、§0の順序「主軸→**構造**→メカニクス」の"構造"を飛ばした。
- **対策:** インカの黄金型の競争骨格を借用(共有リスクプール＋残留者ボーナス＋目玉の早取り)。直接妨害は共有プールに吸収しキングメイク(§1)回避。
- **接合部の衝突:** 目玉の早取り(即時精算)と「見ないで取る(浮上まで価値未確定)」が精算タイミングで衝突→「見ずに奪う=中身は宙吊りのハイリスク奪取」として、衝突を核の見せ場に転化。
- **教訓:魅力的な核はメカニクスから設計を始めさせ、上位の構造(§1)を空白にしがち。§0の順序を守る。複数メカニクスの合成は"接合部"の矛盾を必ず点検。**

## 5. 行き詰まり:なぜインカ型を中止したか(最重要の教訓)

**最終的な問題(設計者の直感):**
「見なかったことの証明」がなされても、このシステムでは**相手の魔力値もノルマ進捗も非公開**。
相手の内部状態が見えない中で「あいつは見ずに取った」という一点だけ公開されても、
**受け手はそれを判断に組み込む土台(文脈)がない** → シグナルが「で?」となり「どっちでもいい」に潰れる。

**根本原因:要素2(公開シグナル)と、インカ型の全面非公開性が、根で噛み合っていなかった。**
- 要素2が輝くには「他の情報はある程度見えていて、その中で"見た/見なかった"だけが未知」という情報環境が要る。
- 全部が霧の中では、一点の光は機能しない。

→ **決定:インカの黄金方針を一旦中止。**

### この行き詰まりの教訓(汎用)
1. **メカニクスには「相性の良い情報環境」がある。** 単体で優れた要素(要素2)も、土台の情報公開度と噛み合わないと死ぬ。要素2は「大半が見えて一点だけ隠れる」環境向きで、全面非公開のインカ型とは逆だった。
2. **独自性の重心がどこにあるかを見極める。** この案の独自性の重心は要素2(見なかったことの証明)にあった。それが土台と噛み合わず機能しないなら、残るのは「PYLに見る/見ないを足したインカの亜種」で独自性が薄まる。**独自性を担う部分が機能しないなら、土台ごと選び直す判断が要る。**
3. **「直感で"どっちでもいい"と感じる」は強い危険信号。** プレイヤーがある選択を「どちらでも大差ない」と感じたら、それは§2-4の偽の選択肢化のサイン。理屈が通っていても、選択が意思決定に影響しないなら死んでいる。

## 6. 最終評価(中止時点)

8. **独自性の重心を特定し、それが機能しないなら土台ごと選び直す。**

## 0. この記録の最大の価値

遅信は理論的検証をすべて通過し、実装まで到達したが、**遊んだら面白くなかった。**

## 1. 遅信の失敗診断

**症状(設計者の言葉):**
「情報が絞られているため、とりあえず通信や浄化連打するだけ。自分が侵蝕されておらず、近くに救難信号が出たら移動するだけ。ジレンマが生まれている気がしない」
「ターン数的に足りそうなら無視、足りなさそうなら助けに行くという判断が単調」

**根本原因:決定論が両刃だった。**
- 決定論的成長は「外挿できる=読みが効く」土台として設計された
- **だが計算できるということは、答えが一意に定まるということ**
- 外挿が「読みの技術」でなく「引き算の作業」になり、行動も一意に決まった
- **悩みとは、天秤が釣り合っているときにしか生まれない。** 遅信の天秤は片方に錘が乗っていた

**構造的欠陥:**
- 各艇が独立していて、自艇を守る資源と他艇を助ける行動がトレードオフでない
- **二つの欲しいものが同じ資源を奪い合っていないと、ジレンマは生まれない**
- 遅延の「わからなさ」は、人間の心の読めなさでなく**単に情報が古いだけ** ―― 引き算で埋まる

> **教訓:情報を絞る/遅らせること自体は、面白さを保証しない。
> 絞られた情報の上で、答えの割れる選択を用意できて初めて機能する。**

---

## 2. 原体験の棚卸し ― 設計の芯を見つける

設計者が「悩ましい」と感じたゲーム体験:
- Hanabi:ヒントを送るべきか、誰に何を送るか
- Nemesis:目標選択、敵対か協力か
- デクリプト:どういうヒントを出すか
- コヨーテ:相手の宣言から自分を推測し、何を宣言するか
- スカル:相手を紛らわせるため、あえて定石を外すか

**共通項の分析:**
- 表面的には「情報系」だが、芯は違う
- **共通するのは「他人の心を読む」こと** ―― 自分の選択の良し悪しが、他人がどう考え動くかに依存し、確定的に計算できない
- **答えを割っているのは、常に「他者という読み切れない存在」**

**遅信との対比:**
- 遅信の他艇は「読み切れない他者」でなく「計算で復元できる決定論的な状態」だった
- 同じ"わからない"でも、**質が正反対**

**さらなる絞り込み(設計者の追加証言):**
- 「受信側と送信側は両方やりたい。強いて言えば受信側」
- 「協力と敵対が切り替わる/混在する方が好み。**敵か味方か確定していないなら、大枠の動きの方向性が大きく変わるから**」
- Nemesisの目標選択は「達成しやすさで選んでいただけ」= そこはジレンマになっていなかった
- **最も固まる瞬間:「自分の敵対的行動を明確に行動に移して、敵であることを今明確にしてよいか否か」**

> **芯の確定:「いつ牙を剥くか」という、正体開示のタイミングの決断。**
> 早すぎれば奇襲の利を失い、遅すぎれば好機を逃す。
> 天秤の傾きは「相手が今どれだけ自分を疑っているか」に依存し、計算できない。
> 全員が同時にこの決断を抱え、互いの牙の予兆を読み合う ―― **送信と受信が一つの決断の中で統合される。**

## 3. 土台の選定

**基底構造の選択:** 「共に守る/築くものを裏切る」(信頼の破壊) vs **「元々競争関係だが表向き手を組み、抜け駆けする」(同盟の解消)** → **後者を採用**
- 打算的な共闘。全員が敵だと分かった上で組む
- 裏切りに道徳的な重さがなく、乾いた緊張になる(§4-1として健全)

**設計者による2つの必須要件:**
1. **裏切りまでに、判断材料になる前兆が得られるターンがあること**(前兆なしの不意打ちは運と同じ)
2. **全員が抜け駆けモードになった後は、速やかにゲームが終わること**(核が尽きたら終わるべき)

**共闘の型:** 相互利益型 vs 共同戦線型 → **相互利益型を採用**
| | 相互利益 | 共同戦線 |
|---|---|---|
| 牙を剥く劇性 | ◎ | △ |
| 前兆システム | ◎ 準備が両義的 | ✕ 行動が均質化し読めない |
| 終了の加速 | ✕ 膠着の危険 | ◎ |
| 核との相性 | ◎ | ✕ |
→ 共同戦線型は**前兆が作れない=核が死ぬ**(核の魅力と欠陥が同じ場所にある=案B・方向iと同型の危険)

**題材:「建てる」vs「潜る」** → **「潜る」を採用**
- 建てる:**積み上げる行為と奪う行為が物理的に逆方向** → 両義性が作れない → 核が死ぬ
- 潜る:「もう一段深く潜る」が**協力の継続とも抜け駆けの準備とも読める** → **両義性が行動に内蔵されている**

**テーマ:崩れゆく遺跡で、盗人たちが略奪する**
- 「なぜ協力するか」= 一人では持ち上がらない(打算)
- 「なぜ裏切るか」= 元々仲間ではない(説明不要)
- 「なぜ急ぐか」= 遺跡が崩れる(時間圧)
- 「なぜ手を離す言い訳が立つか」=「崩れる、もう無理だ」

## 4. 「言い訳」システム(設計者の発案 ― 設計の柱)

> 「裏切る言い訳も用意しておきたい。深海なら酸素の残量が全員バラバラで、『自分は限界だと思ったから浮上した』みたいな」

**言い訳が果たす3つの機能:**
1. **曖昧さを作り、読みを成立させる** ―― 言い訳がなければ「浮上=裏切り」と確定し、読む必要がない。**確定できる前兆は通知であって読みではない**
2. **敗北感の緩衝(§4-1)** ―― 裏切られた側が怒りを宙吊りにでき、裏切る側も心理的コストが下がる。**卓の空気を守りながら背信を促進する**
3. **裏切る側の意思決定を複雑にする** ―― 「言い訳が立つタイミングまで待つ」という駆け引き。**言い訳の有無自体がシグナルの強度になる**

**言い訳が機能する条件:**
- **推定はできるが確定はできない**(完全公開だと曖昧さが消え、完全非公開だと言い訳が万能になり、やはり読みが死ぬ)
- **時間とともに正当性が増す**(利得と正当性が逆方向に働く美しいトレードオフ)
- **事後に検証される**(答え合わせが読みの巧拙を確定させ、学習を成立させる)

## 5. 確定した設計:「共犯」

### コンセプト
崩れゆく遺跡。盗人たちがたまたま居合わせた。互いに仲間ではない。だが重い財宝は一人では運べない ―― だから打算で組む。**全員が、いつ手を離すかだけを考えている。**

### 手番の流れ
1. **財宝が提示される**(必要力、トークン構成 例:[5][3][2][1])
2. 全員が**体力を秘密裏に投入し、同時公開**
3. 合計が**必要力に達すれば成功** → **投入の多い順にトークンをドラフト**(1枚ずつ順に取る)
4. 達しなければ**失敗**(投入した体力は戻らない)
5. または **脱出を宣言して離脱**(以降参加不可。崩落が加速)

### 核となる要素
- **体力の二重性:** 投入にも脱出にも使う。**協力するほど退路が消える**
- **体力の初期値は秘密:** 「限界だ」が真実か嘘か、推定はできるが確定できない
- **必要力は人数で割り切れない値**(3人なら10, 11, 13)→ **均等では届かず、誰かが踏み込む必要がある**(安全均衡の排除)
- **トークンに価値差**([5][3][2][1])→ 「1番手を取るためにいくら出すか」の争い
- **同数時の裁定:** 体力の少ない者が優先(負のFB。「相手の体力を推定して同数なら勝てるか」という読みも生む)

### ノルマ(秘密)
- **絶対ノルマ**(「価値12以上を持ち帰る」)― 他人と両立。協力が純粋に得
- **相対ノルマ**(「他の誰よりも多く」)― 他人と衝突。協力しつつ出し抜く必要がある
- **どちらを持つかは非公開** → 「こいつは協力しても損しない相手か、出し抜く気か」が最後まで読めない
- **ノルマは共同運搬なしには達成できない高さ** ← 協力を構造的に強制する
- **ノルマ未達でも、脱出できれば持ち帰った分が部分点**(最下位の詰み=消化試合を防ぐ)

### 崩落
- 時間圧。**誰かが脱出を宣言すると加速** → 離脱の連鎖が起き、**霧が晴れると同時にゲームが終わりに向かって雪崩れる**(設計者の要件2を構造的に実現)

### 事後の答え合わせ
終了時、**体力の真の残量とノルマを公開。**
「あいつ、まだ体力が9もあったのに『限界だ』と言って抜けたのか」―― **裏切りが確定する瞬間。**

## 6. 独自性

**部品はすべて既存**(秘密同時入札、公共財ゲーム、PYL、裏切り者もの)。独自性は**組み合わせから創発した構造**にある。

1. **投入量が「貢献・前兆・言い訳の布石」の三役を同時に担う**
   - 普通の公共財ゲームでは投入量は単なる貢献度
   - ここでは**同一の数字が三つの意味を持ち、体力非公開ゆえに判別不能に絡まる**
   - 「少なく出した」が貢献の失敗か、抜ける前兆か、体力不足か、**誰にも分からない**

2. **体力が、協力への投資と離脱の手段の両方の原資**
   - 普通の裏切りゲームでは協力と裏切りは別リソースを使う
   - ここでは**協力すればするほど、逃げられなくなる**
   - PYLの構造を「協力行為そのもの」に埋め込み、**累積するのは運のリスクでなく"他人に裏切られるリスク"**

3. **「手を離す」が、離脱であると同時に攻撃である**
   - **不作為が攻撃と同一**であり、しかも**攻撃であることを不作為として言い逃れできる**

> **独自性の重心:「協力すること」と「裏切ること」が、同じ一つの資源(体力)と
> 同じ一つの行為(投入量の決定)の中で、判別不能に重なっている。**
> 普通のゲームは協力と裏切りを別の行動として用意する(協力ボタンと裏切りボタン)。
> この設計では**協力の度合いを調整することが、そのまま裏切りの準備であり実行でもある。**
> 連続的で境界がなく、「今、あいつは裏切ったのか?」が最後まで確定しない。

## 9. 汎用的な教訓(この記録から)

1. **理論的検証を全て通過しても、面白くないことがある。** 遅信は体系のチェックリストを全通過したが、遊ぶと単調だった。**机上の検証には限界がある。**
2. **設計の芯は「自分が実際に固まった瞬間」から取り出せ。** 好きなゲームを列挙し、その中の**どの瞬間**で心臓が動いたかを特定する。「Nemesisが好き」ではなく「敵対を明確化してよいか迷う瞬間」まで降りる。
3. **「答えが計算で一意に出るもの」はジレンマではない。** 悩みは天秤が釣り合っているときにしか生まれない。**答えを割るのは、読み切れない他者。**
4. **穴を塞ぐたびに核が死んでいないか、毎回確認せよ。** この設計では10個の致命傷を修正したが、**そのたびに核5項目を再検証した。**
5. **簡素化は、面白さを削らない。** ドラフト方式への変更で計算が消え、テーマ整合が回復し、**駆け引きはむしろ濃くなった**(トークン価値差)。**複雑なルールは、たいてい設計の失敗を糊塗している。**
6. **「使われない選択肢」は疑え。** 奪いルールはシミュレーションで一度も使われず、検証したら構造的に成立していなかった。
7. **安全均衡を探せ。** 「全員がこう動けば誰も損しない」という均衡が存在すると、駆け引きが死ぬ。

## 0. 出発点 ― 設計者の原体験から核を取る

遅信の失敗(「情報を絞っただけで、結局は計算。ジレンマが生まれない」)を受け、
**設計者自身が実際に固まった瞬間**から核を取り直した。

好きなゲーム(Hanabi / Nemesis / デクリプト / コヨーテ / スカル)を棚卸しし、
「どの瞬間に心臓が動いたか」まで降ろした結果:

> **「自分の敵対的行動を明確に行動に移して、敵であることを今明確にしてよいか」という瞬間。**

これを分解すると、核は3要素:

- **A(加害):** 敵対的行動が、相手に明確な損害を与える
- **B(公然):** それが公然と「敵対」として認識される
- **C(タイミング):** いつ引き金を引くか、自分で選べる

**A・B・C が揃わなければ、核ではない。** これが以降すべての判定基準になった。

> **手法として有効だった:「好きなゲーム」ではなく「固まった瞬間」まで具体化する。**
> 「Nemesisが好き」では足りない。「敵対を明確化してよいか迷う瞬間」まで降りて初めて、設計の核になる。

## 1. 第1のプロトタイプ:遺跡・入札版

### 構造
崩れゆく遺跡。盗人3人。財宝を持ち上げるのに必要な力に、各自が体力を秘密裏に投入し同時公開。
多く出した者から宝をドラフト。**体力は脱出にも要る**(協力するほど退路が消える)。
秘密の目当て5種(蓄財家/出し抜き/目利き/逃げ足/完遂)。

### 実装で発見・修正した問題(13件)
大物が構造的に不可能 / 稼ぎすぎると持ち帰れない / 体力枯渇で「言い訳」が真実になってしまう /
「一番稼ぐ」勝利条件だと協力が無意味 / 分配が軍拡競争を生む / 安全均衡の存在 /
超過処理が複雑すぎテーマから乖離 / 「奪い」ルールが構造的に成立していない /
最下位が詰み消化試合化 / 深度メカニクスが未設計だった / 脱出のダウンタイム / 情報漏洩(同数時の順序から体力が漏れる)…

**修正のたびに核(A・B・C)を再検査した。この規律は正しかった。**

### 結果:「一定の面白さ」で頭打ち
- 投入量の読み合いは成立(**指が止まる瞬間はあった**)
- 出し渋りが前兆として読まれ、フリーライダーの誘惑がチキンレースを生んだ
- **だが核が入っていなかった**

### 診断 ―― 設計者の一言が正確だった
> 「今は、大きな宝は協力して発掘して、バラバラにして、貢献度が高い人から拾い集めるという風に見える」

**この描写に、裏切りが一切登場しない。**
公正な分配の物語であって、背信の物語ではない。

- **A:** 脱出は「離脱」であって加害ではない。残された者が困るだけ
- **B:** 脱出は「敵対の表明」に見えない
- **C:** 「降りるタイミング」であって「牙を剥くタイミング」ではない

> **教訓:核の代用品が面白いと、核が無いことに気づけない。**
> 「投入量の駆け引き」は確かに面白かった。だからこそ、核の不在が隠れた。

## 2. 第2のプロトタイプ:深海・持ち上げ版

### 引き算による再設計
**A・B・Cに奉仕しない要素を、すべて削除した。**
体力回復トークン、目玉★、5種の目当て、価値の異なるトークン、ドラフトの巡回、同数時の判定 ―― 全部切った。

### 構造(核だけ)
沈没船。ダイバー3人。宝石で覆われた箱を持ち上げる。
- 各自が**込める力(0〜4)を秘密裏に決める。公開されるのは合計だけ**
- 力を込めると **「込めた力 + 1」の酸素**を失う(★疲労)
- 酸素は**浮上にも要る** ―― 込めるほど、逃げられなくなる
- **累積が箱の重さに達すれば持ち上がる。** 宝石は持ち上げた比率で分配
- **手を離せる。** 持ち上げた分÷2の宝石を毟り、**箱は闇の底へ落ちる**(累積が全損)
- 箱を掴んだままでは浮上できない。**持ち上げ切るか、手を離すか**

### テーマの完全一致 ―― ここは成功した
| 抽象 | 深海 |
|---|---|
| 体力 | **酸素** |
| 疲労 | **力むと余分に消費** |
| 情報の隠蔽 | **濁った水。互いの姿は見えない** |
| 累積 | **持ち上げた高さ** |
| 加害 | **手を離す。箱は闇へ落ちる** |
| 脱出 | **浮上** |
| 時間切れ | **潮が変わる** |
| 全損 | **溺死** |

**「酸素」という一語が、それまで曖昧だった「体力とは何か」を解決した。**
疲労・二重性・秘匿・時間制限のすべてが酸素で説明できる。**§6のテーマ融合としては理想形だった。**

> **手法として有効だった:メカニクスが要求する条件を列挙し、テーマを逆算する。**
> 7条件(一人では無理/少しずつ/手を離すと全損/一部を持ち去れる/力むと余分に消耗/
> 消耗すると逃げられない/時間制限)を全て自然に満たすテーマとして深海が導出された。

### シミュレーション上は「成功」していた
- 持ち上げ成功 54%(綱渡り)
- **協力 19.9宝石 / 裏切り 19.8宝石 ―― 紙一重**

## 3. 【最重要】なぜ失敗したか ― 2つの誤り

### 誤り1:「期待値を紙一重にすれば悩ましくなる」は間違い

協力19.9 vs 裏切り19.8。**期待値をほぼ同点にすれば決断が悩ましくなる**と考えた。

**実際には ―― 紙一重 = どちらでもいい = 選択が死ぬ。**

> **正しくは:局面によって、正解が鮮明に入れ替わるべきだった。**
> 「今は協力が正しい」「今は裏切りが正しい」が状況次第で明確に分かれ、
> **かつ、今どちらの局面にいるかが読めない**から悩む。

これは遅信の失敗の**裏返し**だった:
- **遅信:** 答えが計算で一意に出る → 悩まない
- **共犯:** 答えがどちらでもいい → 悩まない

**だが ―― 抜けば確実に失敗するので、抜けない。**

必要な力(箱の重さ) ÷ 潜行数 が「全員が毎ターン全力」とほぼ等しく設定されていたため、
**0〜4の選択肢は実質「4」の一択**だった。

> **教訓:「選択肢がある」ことと「選択肢が生きている」ことは違う。**
