## 前提の確認

- **Claude Code は Pro / Max / Team / Enterprise プランが必要。** 無料プランでは使えない
- **ターミナルを自分で開く必要はない。** デスクトップ版は GUI。必要なコマンドは Claude Code が実行し、あなたは許可を押すだけ
- **Node.js は不要**（ネイティブインストーラーがランタイムを同梱）

---

## 手順1. Claude Desktop を入れる

1. `https://claude.com/download` を開く（OSは自動判定される）
2. インストールして起動
3. 左の **「Code」タブ** を開く

---

## 手順2. リポジトリを手元に置く（Claude Code に全部やらせる）

Claude Code に、そのまま日本語で頼む：

```
GitHub の muchkhk/original をこのPCにクローンしたい。
gh コマンドが入っていなければインストールして、認証もして、
そのあとクローンして。
```

- 途中で**ブラウザが開く** → GitHubにログイン → 8桁のコードを貼る → 許可
- 以降、認証は永続する（毎回やる必要はない）
- コマンドを実行する前に Claude Code が許可を求めてくる。内容を見て「許可」を押す

> **⚠️ ここで API キーや Personal Access Token を、チャットに貼らないこと。**
> `gh auth login` のブラウザ認証だけで済む。トークンを手で扱う必要はない。

---

## 手順3. CLAUDE.md を差し替える

クローンできたら、**修正版の `CLAUDE.md`（同時に渡したもの）をリポジトリ直下に置く。**

Claude Code は**セッション開始のたびに CLAUDE.md を自動で読む。**
ここに書いた規律（§0 絶対規律・§2-0 push承認ガード・§7-c 設計中の作品には触るな）が、
以降すべてのセッションに効く。

---

## 手順4. 最初の仕事 ― 指示書 #1

Claude Code に、以下をそのまま貼る。

```
## 指示書 #1 ― 知見MDの嘘を掃除する

【目的】
知見MDは「真実」とされている。古い記述が残っていると、
次のセッションに嘘が伝播する。実装済みの機能が「未実装」と書かれたままの箇所を潰す。

【背景】
seri.html は v1.4（連戦・reveal/lens上書きバグ修正・calcOnlineScores集約・
firebase_rules_seri.json 作成 が入っている）。
しかし『影の競り_オンライン版_引き継ぎ資料.md』には
「同一卓での連戦は未実装」と書かれたまま残っている。
同様の齟齬が他にもある可能性が高い。

【やること】
1. seri.html を読み、実際に実装されている機能を列挙する
2. 以下を実行し、古い記述の候補を洗い出す
   grep -rn "未実装\|未対応\|未検証\|TODO" *.md
3. 洗い出した各項目について、seri.html の実態と突き合わせ、
   「実装済みなのに未実装と書かれているもの」を特定する
4. 『影の競り_オンライン版_引き継ぎ資料.md』を実態に合わせて更新する
5. 更新したMDに、GitHub互換の目次を付け直す（add_toc.py）
6. docs.html を再生成する（build_docs.py）

【触ってはいけないもの】
- seri.html / witch.html / index.html のコード本体（今回はMDの掃除のみ）
- ボードゲーム設計論_理論編.md ／ 実践記録集.md（設計チャット側の管轄）
- 『フリーサイコロ』に関するあらゆる実装（§7-c。まだ設計中）

【完了条件】
- grep で「未実装」と出た箇所が、すべて実態と一致している
- 目次のリンクが機能する
- 変更は push せず、diff を見せて承認を待つ（§2-0）

【報告してほしいこと】
- 実際に見つかった齟齬の一覧（何が「未実装」と書かれていたか）
- seri.html を読んで気づいた、CLAUDE.md §7 に書き足すべき項目
- 懸念（バグの匂い・壊れそうな箇所）
```

## 以降の運用

| やりたいこと | Claude Code への頼み方 |
|---|---|
| バグ修正 | 「seri.html の〇〇が壊れている。直して、版を上げて、diff を見せて」 |
| リリース | 「CLAUDE.md §2 のリリース手順を全部やって」（4箇所の版上げが自動化される） |
| 設計の相談 | **Claude Code に振らない。** 設計チャット（Opus 4.8）で決めて、指示書として渡す |

> **Claude Code は「作りたがる」（CLAUDE.md §6）。**
> **「作らない判断」は、必ず設計チャット側で下す。**

---

## トラブル時

Claude Code に「`claude doctor` を実行して」と頼む。
インストール状態・認証・ネットワーク・設定の破損を一括で診断してくれる。
