## 0. 【共通】まず、プロジェクトナレッジを更新する

新チャットを立てる前に、**Project のナレッジを最新の6ファイルに差し替える**。
古いファイル（`ボードゲーム設計_知見まとめ.md` など統合前のもの）は**削除する**。

```
✅ ボードゲーム設計論_理論編.md          （体系・実例・各論・コンポーネント/影・方法論）
✅ ボードゲーム設計論_実践記録集.md      （案A / 遅信 / 共犯 / 影の賽 ― 4つの失敗）
✅ ボードゲーム制作_プロセス検証知見.md  （設計・検証・秘匿・ルール説明）
✅ ボードゲーム制作_実装技術知見.md      （Firebase・UI・運用の実務）
✅ 影の競り_オンライン版_引き継ぎ資料.md
✅ 魔女の調合_オンライン版_引き継ぎ資料.md
```

**この6本が「真実」。チャットは揮発するが、ここは残る。**

## 2. 💬 チャットA「影の競り ― 実装」の冒頭に貼る

### 現在地

『**影の競り ― 盲目の宝石商たち**』は、**プロジェクト史上はじめて「面白い」に到達した設計**。
（案A・遅信・共犯・影の賽 ―― 過去4作は、すべてこの段階で死んだ）

**核:**
> **「あいつが高く入れた。あいつには何が見えている？ 追うか、降りるか。」**

**ルール骨格:**
```
・原石の価値 = 3〜4つの「面」の合計（🔴色味・💧透明度・✨輝き ／ 5人以上は 💎大きさ）
・各自が見抜けるのは、そのうち1つだけ（＝「目」。ゲームを通じて固定・非公開）
・★ダブりあり ―― 誰がどの目を持つかは分からない。誰の目にも映らない面もある（約56%）
・★見える面は、自分にとって3倍の価値を持つ（私的価値）
・競り上げ式。追うか、降りるか。降りたら戻れない
・落札の瞬間、全面が公開される（＝答え合わせ）
・得点 = 残金 + 競り落とした石の「自分にとっての価値」
```

**人数構成（実装済み・2〜6人）:**

| 人数 | 面 | 石 | 資金 | 「誰の目にも映らない面」 |
|---|---|---|---|---|
| 2人 | 3 | 6 | 30 | 100% |
| 3人 | 3 | 6 | 30 | 78% |
| **4人** | **3** | **8** | **34** | **56%（標準）** |
| 5人 | **4** | 10 | 38 | 77% |
| 6人 | **4** | 12 | 38 | 62% |

### 実装済み（seri.html v1.3）

- **ソロ（NPC対戦・2〜6人）** … NPCは履歴から「あいつの目」を推定し、「自分と同じ目か？」で情報価値を割り引く。**追随はしない**（呪いが悪化するため）
- **オンライン対戦（Firebase）** … 部屋作成・参加・観戦。transaction による原子的更新、権限の二重ガード、生存報告12秒＋30秒タイムアウト、withRetry、エラーの赤バナー
- **観戦モード** … 全員の目・全面・**接続状態**・手番・進行フェーズ（＝「異常が一目で分かる」）
- **版の食い違い検出** … 卓の版と自分の版が違えば赤バナー
- **キャッシュ自己修復** … Service Worker 解除 ＋ no-store で版を照合 ＋ `?v=` で別URLへ自動遷移
- **UI** … キーボード操作（Space=追う / Esc=降りる）、手番通知（音＋タブ点滅）、**誰がいくらで降りたか**の表示、用語ツールチップ、IME保護、ルール説明（専門用語ゼロの概要→動詞で書いた流れ→詳細→用語集）

### 数値検証の結論（済み）

- **割引率に内点均衡あり**（支配戦略なし）。慎重すぎ（0.6）は勝率12.8%で罰される
- **負の成分は必須** ―― 無くすと呪い率が3.6%に崩壊する
- **私的価値 TASTE=2 が適正**（1→呪い20% / 2→12.7% / 3→8.7%）
- **★「他人の入札に追随すると負ける」**（追随0 → 勝率32.4% ／ 追随1.0 → 13.7%）。
  **これは欠陥ではなく学習曲線。**「勝者の呪い」としてルールに明記済み
- 純粋な共通価値オークションは競りが死ぬ ―― **私的価値が必須だった**

### 未検証・次にやること

- [ ] **人間同士のオンライン実プレイ**（最重要。NPCでは核を測れない）
- [ ] **2回目・3回目でも面白いか**（最適解が固まらないか）
- [ ] 6人卓の勝率の幅が 13.6pt とやや大きい（他は7〜9pt）。運が重くないか
- [ ] Firebase のセキュリティルール（`boadgame2`）が、テストモードのまま放置されていないか

### 既知の限界（正直に）

- **ホストは全原石を知りうる**（生成者だから）。UIには出さないが localStorage を見れば分かる
  → **厳密にやるなら「遊ばない人がホストになって観戦する」**（実装ゼロで最も安全）
- `priv/{席}` は認証なしでは他人も読める（RTDBの制約）。身内プレイ前提の「最小対処」
- **ホストが離脱すると、以降の原石が開けられない**（ホストが再入場すれば復帰）

## 3. 💬 チャットB「ボードゲーム設計論」の冒頭に貼る

### この体系の中心概念：**「影」**

> **影 ＝ 形は伝えるが、中身は伝えない情報層。**
> **情報を粗い層と細かい層に分け、粗い層だけを公開する技法。**（理論編・第4部§3）

**3種類ある:**

| | 誰が濃さを決めるか | 例 |
|---|---|---|
| **静的な影** | 設計者（土台を作るが、決断は生まない） | 海底探険のレベル / 危険度の帯 |
| **動的・発信型** | プレイヤーが自ら晒す | 麻雀の副露・リーチ / **影の競りの入札** |
| **動的・取得型** | プレイヤーが「どの層を見るか」選ぶ | 物理では不可能＝**デジタル固有** |

### 【最重要原理】4つの失敗が生んだもの

> ## **「情報を集める行為」は、選好のシグナルにならない。**
> ## **選好を信頼できる形で伝えるのは、「コストを払う行為」だけである。**

| 行為 | 何のシグナルか |
|---|---|
| 調べる・観測する | **不確実性**（「私はまだ分からない」） |
| **賭ける・入札する・支払う** | **選好**（「私はこれに価値を認める」） |
| 麻雀の副露・リーチ | **コミット**（資源を払った／後戻りできない） |
| **調べた上で"手放す"** | **選好の負のシグナル**（レモン市場・逆選択） |

**これが『影の競り』を成立させた。**（＝入札は、コストを払う行為だから）

### 4つの失敗と、その死因

| | 核 | 死因 |
|---|---|---|
| **案A** | 見なかったことの証明 | 核と土台の**情報環境**が不一致 |
| **遅信** | 情報の鮮度の資源化 | **答えが計算で一意に出る** |
| **共犯** | 敵対を明確化するタイミング | **答えがどちらでもいい** |
| **影の賽** | 調べれば奪われる | **核の前提が、論理的に成り立たない** |

> **遅信・共犯は「バランスの病」。影の賽は「論理の病」。**
> **バランス調整では絶対に直らない。**

### 保留中のアイデア（未検証）

**① シュレディンガーのサイコロ**（理論編§6-16）
> 出目が全て異なる6面ダイス。全面が非公開。**振るたびに1面ずつ明らかになる。**
> **未知の面こそが価値であり、振ることは、その価値を消費すること。**
> **決定的な空白：「振る」ことに、なぜ緊張があるのか？**
> （A:得点＝PYL ／ B:全員に公開＝影の賽の死因の再来 ／ **C:自分だけ見える＝レモン市場**）

**② フリーサイコロ**（理論編§6-17・**最有力**）
> 6面すべて白紙のダイス。**使った数字が、その面に永久に刻まれる。**
> **自由を使うことが、自由を消費すること。**
> **共通ダイスで「面の番号」を振る**（＝運は共有、判断は個人。紙ペンの構造）
> **土台が未決定**（エリアマジョリティ vs 共有の注文）。§0の設計順序を守ること

### 未解決の問い

- [ ] フリーサイコロの土台をどうするか（**数字は「強さ」ではなく「種類」でなければならない**）
- [ ] 「自由の減り方」を、濃い決断の帯に保てるか（序盤＝全能で自明／終盤＝硬直で決断消滅）
- [ ] §5-2 の「操作 × コンポーネント」行列の空白（サイコロ×視点化 など）

### 議論のときの規律

1. **理屈より、設計者が固まった瞬間を優先する**（共犯の最大の教訓）
2. **「核が弱い」という検証結果が出たら、まず「核を実装したか」を疑う**（影の賽で実際に踏んだ）
3. **一般化できる学びは、その場でMDに書き戻す**
