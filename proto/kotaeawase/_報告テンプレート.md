# 報告テンプレート（『こたえあわせ（仮）』作業報告 用）

> このファイルは**生成物ではなくテンプレート**。コピーして使うこと。直接編集しない。
> CLAUDE.md §9-g「指示書への報告は、チャットだけでなくファイルとしても作成する」に対応する定型。
>
> **使い方：**
> 1. このファイルを `proto/kotaeawase/報告_<作業名>_<版>_<日付>.md` としてコピーする
>    （例：`報告_こたえあわせv0_8_○○対応_2026-08-01.md`）。
> 2. `【…】` の部分を実際の内容に置き換える。使わない節は削除してよいが、
>    指示書に【報告してほしいこと】が明記されている場合は、その項目を必ず対応する節として残す。
> 3. 実装物（コード変更）と同じコミットに含める（同じ場所にコミットする＝§9-g）。
> 4. 目次（TOC）は付けない（`proto/`配下の報告書はこれまで一貫して無し。知見MD（`docs/*.md`）とは扱いが異なる）。

## 6. push・bundle

CLAUDE.md §2-0（push前に必ず人間の承認を取る）に従う。

- 承認前：「コミット済み・push未実施」または「未コミット」の別と、理由を明記。
- 承認後にpushした場合：コミットハッシュ・pushしたブランチ・本番URLでの版バッジ確認結果。
- 403等でpushできない場合：全履歴入りbundle（`kotaeawase-<VER>-full.bundle`）を生成し、
  `git bundle verify` の結果と適用手順（`git fetch ... / git checkout ... / git push ...`）を記載する。
