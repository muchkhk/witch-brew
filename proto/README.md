# witch_solo_proto — タイプ開示NPCソロ試作（人間テスト用）

指示書「witch ソロプロトタイプ実装（タイプ開示NPC・ローカル版）v1」に基づく使い捨てプロトタイプ。
**kill/keep判断が通るまで witch.html 本体には一切手を入れていない。**

## 起動方法

`witch_solo_proto.html` をブラウザでダブルクリックして開くだけ（Firebase不要・サーバ不要）。
Chrome/Edge/Firefoxで動作確認済み。

## 遊び方

1. 名前を入力し、NPCタイプを2体選ぶ（ミラ＝世話焼き／ゾラ＝欲張り／ルゥ＝気分屋、詳細は起動後の画面参照）
2. 「タイプ表示」トグル（初期値ON）で、NPCの性格ラベル・説明を隠す/見せるを切り替えられる
3. マルチと同じ流れ：6儀式・切り役ローテーション・チーム合計のみ公開（B方式）
4. 自分が切り役の儀式では帯の区切りをクリックして「この位置で分ける」、
   選ぶ番では坩堝をクリックして選択。NPCの手番は自動で進む（考え中演出あり）
5. 終了後、通算冴え(pct)・称号・全儀式のログが表示される

## NPCタイプとパラメータ（v3計測済み・変更禁止）

| タイプ | 表示名 | λ | q | w |
|---|---|---|---|---|
| 世話焼き | ミラ | 0.65 | 1 | 0.5 |
| 欲張り | ゾラ | 0.8 | 1 | 0.5 |
| 気分屋 | ルゥ | 0.65 | 2 | 1.0 |

τ=0.3固定（気まぐれバイアス方策族、`sim/witch_solo_npc/`の計測結果参照）。
気まぐれカードは毎ゲーム引き直し、採点には一切関与しない。

## 既知の制約

- **3人固定**（プレイヤー1＋NPC2）。人数選択機能はない
- **上級モード（秘伝枠）は未対応**。通常モードのみ
- **見た目は最低限**。使い捨てプロトタイプのため装飾・アニメーションはほぼ無い
- 推定エンジン（信念更新）はブラウザのメインスレッドで同期実行される。手番ごとに一瞬の
  計算時間が発生する（体感数十〜数百ms程度。詳細は結果報告を参照）
- リロードすると進行中のゲームは失われる（永続化なし）
- NPCが2体とも同じタイプは選べない仕様（意図的な制約）

## 同値性検証

`verify_equivalence.js` を Node.js で実行すると、`gen_reference_traces.py`（Python版
`sim/witch_solo_npc/solo_npc_sim.py` + `solo_npc_sim_v3.py` を使用）が生成した
`reference_traces.json`（3ケース・6儀式フル）を、`witch_solo_proto.html` に実際に埋め込まれた
JSエンジンで再生し、信念更新の結果（候補数の推移）がPython版と一致するかを確認する。

```bash
node verify_equivalence.js
```

検証ログ: `verify_final.log`（**9/9系列で完全一致**を確認済み）。
