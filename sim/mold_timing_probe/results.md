## 総合判定：FAIL（`overall_pass = false`）

事前登録§6は「以下のいずれか1つでも発動したら案Aをkillする」というOR条件であり、**K-A2が発動した**ため、他の3条件の結果によらず案Aはkillとなる。

## kill条件ごとの判定（事前登録§6の文言のとおり）

### K-A1（待ち屋支配）：発動せず（`fires = false`）

判定規則：「3曲線すべてで、X∈{S-early, S-th65, S-mold}のいずれについても、対応差CI95下限(X − S-wait) ≤ 0」が成立したら発動。

| 曲線 | X | CI95下限(X − S-wait) |
|---|---|---|
| C1 | S-early | −0.2034 |
| C1 | S-th65 | 0.4810 |
| C1 | S-mold | 0.4515 |
| C2 | S-early | 0.2304 |
| C2 | S-th65 | 0.7357 |
| C2 | S-mold | 0.7297 |
| C3 | S-early | −0.0232 |
| C3 | S-th65 | 0.7824 |
| C3 | S-mold | 0.7527 |

9値中、正の値（>0）が7つあり「すべてが≤0」を満たさないため不発動。

### K-A1'（情報無価値）：発動せず（`fires = false`）

判定規則：「3曲線すべてで、X∈{S-wait, S-th65, S-mold}のいずれについても、対応差CI95下限(X − S-early) ≤ 0」が成立したら発動。

| 曲線 | X | CI95下限(X − S-early) |
|---|---|---|
| C1 | S-wait | 0.1864 |
| C1 | S-th65 | 0.6713 |
| C1 | S-mold | 0.6422 |
| C2 | S-wait | −0.2481 |
| C2 | S-th65 | 0.4910 |
| C2 | S-mold | 0.4859 |
| C3 | S-wait | 0.0078 |
| C3 | S-th65 | 0.7936 |
| C3 | S-mold | 0.7644 |

同様に「すべてが≤0」を満たさないため不発動。

### K-A2（曖昧帯が装飾）：**発動（`fires = true`）** ← これ単独でkill

判定規則：「各ダイス個体について『鋳型事後P(H|そのダイスの開示1..t)が閉区間[0.2, 0.8]に入っているtの個数（t=1..6中）』を数え、その全個体中央値が2未満」。

- 全ダイス個体（100,000ディール×2 = 200,000個体）の中央値 = **0.0**（閾値「2未満」に該当）
- 解釈：鋳型H={4,5,6,7,8,9}とL={1,2,3,4,5,6}は3値ずつが排他的（1,2,3はLのみ・7,8,9はHのみ）で3値が共有（4,5,6）のため、開示された最初の面の値が排他的値である確率はちょうど1/2。この構造上、半数以上のダイスは開示ステップ1の時点で鋳型が確定してしまい、「曖昧なまま賭けの判断を迫られる」局面がほとんど生まれない。事前登録が狙っていた「曖昧帯での駆け引き」は、この価格・鋳型設定では実質的に存在しなかった。

### K-A3（隠れ変数が装飾）：発動せず（`fires = false`）

判定規則：「3曲線すべてで、対応差CI95下限(S-th65 − S-blind) ≤ 0」が成立したら発動。

| 曲線 | CI95下限(S-th65 − S-blind) |
|---|---|
| C1 | 0.0161 |
| C2 | 0.0598 |
| C3 | 0.0256 |

3曲線とも正（>0）であり「すべてが≤0」を満たさないため不発動。鋳型（隠れ変数）を正しく推論するモデル（S-th65）は、鋳型相関を無視する周辺分布モデル（S-blind）を、統計的に有意に上回っている。

## 仕様欠落と解決（詳細は`事前登録_鋳型工程1_kill基準.md`§9）

実装開始前に、S-early/S-wait/S-moldの「賭けの方向」を決める事後モデルが事前登録に明記されていない欠落を発見し、実装を一旦停止して報告した。ユーザー確認の結果、S-blind以外の全戦略は真の生成モデル（鋳型ベイズ）の勝ち事後を方向決定にも流用する、という解釈で実装した（詳細は事前登録§9）。
