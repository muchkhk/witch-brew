## §3 新指標 M2'（見立ての改善量）の定義 ―― 指示書からの転記（echo）

各ディールについて、開示前の見立てで勝敗を予想し、開示後の見立てで予想し直したときの「的中確率の改善量」G を求める。

1. 開示前の予想：eq_pre > 0.5 なら「自分が勝つ」、eq_pre < 0.5 なら「相手が勝つ」、eq_pre = 0.5 ちょうどならコイン投げ（的中確率0.5として扱う）。
2. 開示前予想の真の的中確率 A_pre を、開示後の真の勝率 eq_post で評価する：eq_pre>0.5なら A_pre=eq_post、eq_pre<0.5なら A_pre=1−eq_post、eq_pre=0.5なら A_pre=0.5
3. 開示後予想の的中確率 A_post = max(eq_post, 1 − eq_post)
4. G = A_post − A_pre （定義上 G ≥ 0）

**M2' = 全ディールにわたる G の平均値。** 一次統計量は平均のみ（中央値は不使用、凍結）。

## §4 判定規則v2 ―― 指示書からの転記（echo）

**手順0（検品v2）：** F+ が「M2'（点推定）≥ F0のM2'（点推定）×1.5」かつ「M3 ≥ max(2.0, F9のM3×1.5)」を両方満たすこと。
満たさない場合、指標の再修正は行わず、全数値を添えて停止・報告し、アプローチ自体の再検討をdesign chatへ差し戻す。

検品合格後：
- **Kill-L**：M2' の95%CI下限が F0 の M2' 点推定の1.5倍未満 → kill
- **Kill-H**：M3 の95%CI上限が max(2.0, F9のM3×1.5) 未満 → kill（v1と同一の値を使用）
- **Pass**：両killを回避し、かつ M2' の点推定が F+ の M2' の50%以上

タイブレーク：M3最大 → 同点ならM2'最大 → なお同点なら FC1＞FC2＞FB＞FA。

## 候補4本のKill-L/Kill-H/Pass判定（機械的適用）

Kill-L閾値 = F0のM2'点推定 × 1.5 = 0.025556 × 1.5 = **0.038334**
Kill-H閾値 = max(2.0, F9のM3×1.5) = **2.0**（v1のM3・CIをそのまま使用）

## 実装判断メモ（v2固有）

1. **配り直しの回避方法**：v1の`simulator.py`をモジュールとして`import`し、`FOUNDATIONS`辞書・`DEAL_SEED`・
   `SeededRandomChoice`をそのまま再利用した。これにより「同一シードでの再現」ではなく「同一コードパスの
   再実行」となり、論理的にv1と完全一致することが保証される（実測でも不一致0件）。v1のロジックへの変更は
   一切行っていない。
2. **「反転」の定義（報告項目3用）**：M2'自体はeq_pre=0.5のケースも自然に扱う連続的な量だが、
   「反転が起きたディールの割合」という離散的な内訳を出すために、**v1の旧M2と全く同じ反転定義**
   （eq_pre=0.5ちょうどの場合はeq_post≠0.5を反転とみなす、を含む）を流用した。指示書の§3「性質」の
   記述（反転時 G=|2eq_post−1|）はeq_pre≠0.5の場合の式であり、eq_pre=0.5ちょうどの場合はG=|eq_post−0.5|
   という別式になる（F9はこの後者に該当し、G=0.5固定になる）。両ケースとも「反転」カテゴリに含めて
   平均を取っているため、avg_G|flip はこの2種類の式が混在した値である。F9以外では eq_pre=0.5ちょうど
   となるディールはごく少数（またはゼロ）と見られ、実務上の影響は小さいと判断した。
3. **ブートストラップの再シード単位**：v1同様、「土台ごとに独立に同一定数シード(20260800)で再シード」
   という解釈を維持した（v1 results.mdの実装判断メモ#2と同一の解釈）。
4. **M3の扱い**：指示書の指定通り、v1の`results.csv`から値をそのまま読み込んで転記し、再計算していない。

凍結項目（§3・§4）そのものの変更が必要だと判断した箇所はない。
