## 総合判定：PASS（`overall_pass = true`）

事前登録§6のkill条件（OR——いずれか1つでも発動でkill）は、K-D1'・K-D1・K-D4・K-D3の**いずれも不発動**。§0-2の限定に従い、Passの含意は「移動選択なし（固定順）・覗き非公開・賭け±1固定の条件下で、演繹コアが正の利ざやを持つ」ところまで。

**注目点：K-D4はCI95下限0.033と、他の条件（K-D1'/K-D1: 0.67〜0.95、K-D3: 0.85）に比べて著しく僅差でのPassだった。** 演繹チャネル固有の市場価値は「存在するが薄い」——詳細は下記。

## kill条件ごとの判定（事前登録§6の文言のとおり）

### K-D1'（情報の構造的無価値）：発動せず（`fires = false`）

判定規則：条件A・c=0で、選択集合3戦略すべてについて対S-free勝率のCI95下限 ≤ 0.5 → 発動。

| 戦略 | CI95下限（c=0） |
|---|---|
| S-peek-imm | 0.9492 |
| S-peek-marginal | 0.9238 |
| S-peek-prop | 0.6726 |

3戦略とも0.5を大きく上回るため不発動。無料でも情報は勝率に確実に換金される。

### K-D1（市場不在・主判定）：発動せず（`fires = false`）

判定規則：条件A・c=0.5で、選択集合3戦略すべてについて対S-free勝率のCI95下限 ≤ 0.5 → 発動。

| 戦略 | CI95下限（c=0.5） |
|---|---|
| S-peek-imm | 0.7236 |
| S-peek-marginal | 0.8477 |
| S-peek-prop | 0.4955 |

S-peek-propのみ0.5をわずかに下回る可能性がある範囲（下限0.4955）だが、imm・marginalは明確に0.5超のため「すべて」の条件を満たさず不発動。市場は価格0.5でも実在する。

### K-D4（演繹は市場に寄与しない）：発動せず（`fires = false`）―― 最も僅差のPass

判定規則：X*（条件A・c=0.5の点推定が最大の戦略）について、対応差CI95下限( exc_A(X*) − exc_C(X*) ) ≤ 0 → 発動。

- 選ばれた X* = **S-peek-marginal**（条件A・c=0.5の点推定 0.8500 が3戦略中最大）
- exc_A(X*) 点推定 = 0.34996　／　exc_C(X*) 点推定 = 0.31371
- 対応差 CI95: 下限 **0.03341** ／ 上限 0.03899

CI95下限は0（発動閾値）をわずかに上回るのみで、Passはしているが**マージンが極めて小さい**。演繹（相異制約）が市場に寄与する分は統計的に有意ではあるが、量としては小さい——市場価値の大半は「隠し面ダイスなら何にでもある即時チャネル(imm)」由来であり、本案固有の演繹(prop)由来の追加価値は薄い。

### K-D3（ロンデルは装飾）：発動せず（`fires = false`）―― 大差のPass

判定規則：Y*（選択集合{imm, marginal}の中で条件A・c=0.5の点推定が最大の戦略）について、対応差CI95下限( exc_A(Y*) − exc_B(Y*) ) ≤ 0 → 発動。

- 選ばれた Y* = **S-peek-marginal**
- exc_A(Y*) 点推定 = 0.34996　／　exc_B(Y*) 点推定 = **−0.5**（条件Bでは対S-free勝率が厳密に0.0 = 常に負け）
- 対応差 CI95: 下限 **0.84774** ／ 上限 0.85212

大差でPass。ただし事前登録が明記するとおり、**この非発動を「締切圧の実証」と読んではならない**。理由は次節。

## 条件Bに関する重要な構造的知見（実装前に導出・検証済み）

条件B（ランダム順）では、賭けは「今ラウンド問われる位置がどれか」を知らない状態でコミットされる（§3-2）。この場合、事前登録が明記する事後式「残Hカウント／未公開数」は**公開情報のみで決まる量**であり、これは近似ではなく、"対象位置が不明なまま単一の賭けを確定させる"という条件Bの構造から数学的に強制される帰結である（証明はexchangeabilityの一般論、`simulator_process2.py`冒頭のdocstring・末尾コメント参照）。

結果として、**条件Bでは覗きは当該ラウンドの賭け的中率を一切改善しない。** 覗き戦略の得点は理論上「S-freeと同一の賭け結果 − 自分の覗き支出」に厳密に一致するはずであり、実装前に導出したこの予測を、実際のシミュレーションで**全件・厳密一致（100%）で確認済み**（自己検証(c)の追加項目として全2000ゲームで検証、本番実行でも`dud_rate_condB`・`price_grid`の数値が矛盾なく整合）。

この帰結により、条件B・c=0.5〜1.0での対S-free勝率は**imm・marginalとも厳密に0.0**（覗き回数がほぼ必ず1回以上発生し、価格>0なら必ずS-freeに劣後するため）。K-D3はこの「ほぼ確実に負ける条件B」と条件Aを比較する検定であり、**Passすることは構造的にほぼ保証されている**——事前登録自身がこの点を織り込み済みで、非発動の過大解釈を明示的に禁止している。

### 覗き対象分布（条件A・C固定順は構造上自明・条件Bは「不発率」で代替報告）

条件A・Cでは、S-peek-imm/marginalは定義上「当該ラウンドの賭け対象」のみを覗くため、**位置5・6を選ぶことは構造的にあり得ない**（peek_rate_by_roundが0〜1で位置1〜4のみに対応）。S-peek-propは定義上、位置5・6のみを覗く（他の位置を一切覗かない）。実測でもこの構造どおりであることを確認済み。

条件Bでは「問われない位置」という概念自体が存在しないため、代わりに**不発率**（覗いた位置がその回の実際の対象と一致しなかった割合）を報告する：

| 戦略 | 不発率 | 覗き機会数 |
|---|---|---|
| S-peek-imm | 0.7834 | 326,683 |
| S-peek-marginal | 0.7935 | 215,441 |

約79〜80%の覗きが「対象と一致しない」。ただし上記構造知見のとおり、対象と一致してもしなくても賭けの的中率には無関係（覗きの価値そのものが条件Bではゼロ）。

## §8の3項事前見積もりとの突き合わせ

事前登録§8は「工程の結果がFAILの場合、死因の所在を突き合わせる」としているが、本工程は総合PASSのため死因の局在化は不要。ただしK-D4の僅差Passは、§8第1項（無料推定との乖離）が示唆した「演繹の追加価値は薄い」という懸念が、実測でも定量的に裏付けられた形になっている。
