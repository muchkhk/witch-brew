## 2. 手法

### 2-1. NPC方策のパラメータ化

- **λ（混合比）**：塊選びの効用 = λ×自分の得点 + (1−λ)×残りメンバーへの信念期待値。
  切り役は自分の坩堝を選べないため、λによる自己利益/チーム利益の混合は塊選びのみに適用し、
  切り役はwitch.html自身の`bestForCut`/`omniMax`の思想（信念期待値での最適配置）を踏襲した
  （2026-07-15 design chat裁定で承認済み）
- **τ（揺らぎ）**：効用に対するsoftmax温度。τ→0で決定的、大きいほど一様に近づく
- グリッド：λ∈{0.5, 0.65, 0.8, 0.9, 1.0} × τ∈{小=0.5, 中=2.0, 大=6.0} の15組（変更なし）

### 2-2. 推定エンジン

観測可能な行動（帯・切れ目・誰がどの塊を選んだか、チーム合計点）のみを入力とし、816通り全手札
仮説について対数尤度を逐次更新する。チーム合計の取り込みは、各プレイヤーの得点分布を信念で
重み付けし畳み込む近似（joint厳密解ではない。2026-07-15 design chat裁定で承認済み）。
候補数の定義（最尤仮説の尤度の1/20以上）は事前登録どおり変更していない。

### 2-3. 計測B の比較条件

一貫方策プレイヤー = λ=1.0・τ→0（自己利益に忠実）、無作為プレイヤー = 効用に依存せず
ほぼ一様選択（τ=1e6で近似）。NPC切り役下での対象プレイヤーの個人点合計（6儀式）を比較。

## 4. 候補数曲線の分析

### 4-1. τが支配的なパラメータで、λの影響は小さい

候補数曲線・A3的中率とも、**τ（揺らぎ）の水準でほぼ決まり、λ（自己利益/チーム利益の混合比）
による違いはごく小さい**。15組の中でλを固定してτだけ動かすと候補数中央値が数十〜数百倍
変化するのに対し、τを固定してλだけ動かしても数通り〜十数通りの違いしかない。

### 4-2. A1/A2とA3が同時に満たせない構造的な緊張関係

- **τ=大(6.0)**：A1・A2は通過（6儀式後も候補55〜63通りと広い＝早期特定されない）が、
  A3の的中率は13〜14%と低い（行動が揺らぎすぎて、推定エンジンが1枚に絞り込めない）
- **τ=小(0.5)**：A3の的中率は55〜67%と高く**5組すべてで基準を達成**（行動が一貫していて読みやすい）が、
  A1・A2は大幅に不合格（6儀式後には候補3〜4通りまで絞れてしまい、「早すぎる特定」に該当）
- **τ=中(2.0)**：両者の中間だが、どちらの基準にも届かない（帯に入らずA1/A2不合格、
  的中率23〜28%でA3にも届かない）

つまり本グリッド内には、**「早すぎず読める」ことと「最終的に高確度で言い当てられる」ことを
両立するτは存在しない**（τ=小はA3単独では達成するが、その代償としてA1/A2を完全に失う）。
より細かいτの探索（例: 0.6〜1.0の範囲）で両立点が見つかる可能性は残るが、
少なくとも今回の3水準グリッドでは全滅した。

### 4-3. B1は全条件で頑健に成立

一貫方策プレイヤーは、無作為プレイヤーに対してNPC切り役下で常に大きく（+94%〜+142%）
上回っており、95%信頼区間も0をまたがない。**「読まれる甲斐」自体は、τ・λの設定に依らず
頑健に成立する。**（意外性のある知見ではなく、λ=1.0(一貫)がλ=0.5,τ=無限大(無作為)より得点で
勝るのはほぼ自明だが、効果量が非常に大きい点は特筆に値する）

## 6. 計算時間の実績

- 素朴実装：0.14秒/試行 → 信念正規化のキャッシュ化・ヒストグラム構築のnumpy化で **0.029秒/試行**
  （約5倍高速化）
- 実測：全15組×計測A/B/C（旧版）+マルチ基準、n=2000、**総実行時間 約63分**
  （マルチ基準74.5秒、各組は計測A約59秒・計測B約123秒・計測C約64秒）
- 計測Bが最も重い（一貫・無作為の2条件×n=2000のため、他の測定の約2倍）
- 今後スイープを拡張する場合、1試行あたり約29ms、1計測(n=2000)あたり約58〜125秒が目安

## 7. 実行方法

```bash
cd sim/witch_solo_npc
python solo_npc_sim.py --smoke                          # 動作確認(n=20、数分)
python solo_npc_sim.py --sweep --n-trials 2000           # 本番スイープ(約60〜70分)
python run_descriptive_c.py sweep_results.json           # 記述的C(A・B通過組があれば)
```

依存: Python 3.10, numpy。決定論性: `MASTER_SEED=20260715` から派生した
`numpy.random.Generator` のみを使用。
